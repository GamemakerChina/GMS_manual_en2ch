函数

这部分解释了在GameMaker: Studio中函数是什么。

函数的基本定义是这样的：函数有输入（自变量）和输出（因变量），输出用某种方式对应着输入。 在 GameMaker: Studio中，你可以使用函数作为 程序 的一部分，作用于游戏的某些东西。并且 GML (GameMaker Language) 中有大量可用的函数，参考手册部分将作详细解释。

GML函数结构是，一个函数名，后面是括号之内的输入的参数，参数用逗号分隔（如果没有参数，仅仅用括号就可以了）。下面是函数结构的轮廓：

<function>(<arg0>, <arg1> ,... <arg15>);


基本上有两类函数 - 首先是大量的内置函数，控制游戏的所有方面，其次是任何你在游戏中定义的脚本也可以当作函数使用（但不总是这样）。

某些函数有返回值可以用在表达式中，而其它的只是简单的运行命令，注意不要将函数用在一个赋值的左手边。比如，你不能这样写：

instance_nearest(x, y, obj).speed = 0;


因为返回值是一个实数，这种时候你必须将整个函数用括号包围起来（更多信息请看 访问其它实例的变量）， 所以代码实际上应该这样写：

(instance_nearest(x,y,obj)).speed = 0;
