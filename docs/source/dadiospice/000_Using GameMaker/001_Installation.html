安装及系统需求

安装个人使用的 GameMaker: Studio。

如果您的 GameMaker: Studio 版本 不是 从 Steam 购买的，您需要运行 GMStudio-Installer.exe 程序并且根据屏幕上的提示安装。您可以选择任何您喜欢的目录安装本程序，但建议您选择的默认的位置进行安装。一旦安装完成后，在开始菜单中你会找到一个新的程序组，你可以在这里找到运行 GameMaker: Studio 并且阅读帮助文件。

安装 GameMaker: Studio 的同时会在您的计算机上安装许多images (图像)、sprites (精灵)和 sound effects (音效)这些都可以在您开始制作游戏时作为 "占位符" 使用(但是他们 不 能使用在任何成品里)。并且在教程里还有很多示例游戏，以及更复杂的实例演示。这些可以在 GameMaker: Studio 的安装文件夹里找到，也可以通过运行 GameMaker: Studio 窗口并且在IDE (集成开发环境)里找到。 (从 帮助菜单)。

第一次运行 GameMaker: Studio 主窗口的右边将会显示一个教程，这个教程将会一步一步教您如何快速地创造第一个游戏，使得您可以尽快地学习使用 GameMaker: Studio 。这个教程可以跳过，但是如果您没有使用过 GameMaker: Studio (或者任何以往版本的 GameMaker) ，强烈建议您学习它，通过它了解游戏的制作过程。如果您跳过了教程，可以随时从 帮助 菜单再次打开它。

注意: 如果您是在 Steam 购买的 GameMaker: Studio，程序会自动安装，可以选择创建开始菜单及桌面快捷方式。您可以点击这些快捷方式或者通过 Steam 客户端 运行本软件。

基本需求

任何年龄小于5年的电脑运行 GameMaker: Studio 都没有问题，但是您需要足够的空间安装本软件以及其他的模块。

Windows

GameMaker: Studio 需要一个运行 Windows XP、Vista、7 或更高版本系统的电脑，一个兼容 DirectX 9 (或更高版本) 、显存 至少 32MB的显卡。需要至少 1024x768像素 和 65536 (16位) 色 (最好是32位真彩色)的屏幕。一个兼容 DirectX 9 的声卡或者集成语音芯片。确保您的电脑安装了最新的驱动。

GameMaker: Studio 设计和测试游戏时需要在计算机上安装 DirectX 9或更高版本(您可以从以下网站: http://www.microsoft.com下载DirectX 9.)，对内存要求会很高 (至少128MB或更多，取决于操作系统)。内存需求很大程度上取决于游戏类型以及所使用的资源。

注意: 尽管现代的电脑安装了 DirectX 10 或者更高版本但这并不意味自在您的机器上也安装了 DirectX 9，所以您仍然需要下载它。

请注意，Steam用户还可以添加一个额外目标为 Steam 创意工坊。当创建和测试您的游戏，额外的模块将会以标准的windows模块运行，然后当您创建最终exe可执行程序时将显示 直接提交到steam 的额外选项。更多信息可以点击 这里.

Windows 8

专业版 GameMaker: Studio 带有 Windows 8 本地和 JavaScript 目标平台作为额外的目标窗口。这些目标需要一个 Windows 8 操作系统来运行 (和测试)。

您还需要确保您安装有 Microsoft .Net Framework 3.5 (.Net 4 内置在Windows 8里，但是您可能也需要之前的版本)，可在 here, 下载，尽管当你在 Windows 8 上安装 GameMaker: Studio 的时候它会自动完成这一部分。

另一件事是需要一份 Microsoft Visual Studio 的拷贝，可以在 这里 找到。您必须只使用 VS 2012、2013 (或者 Express 版)，如果您使用旧版的VS，它不会与该模块一起工作。这个版本的VS支持创建 Windows 应用商店 JavaScript 软件所需要的 *.pfx 文件，并且需要 Developer Key(开发者秘钥) 一并提交到 Windows 应用商店。

GameMaker: Studio可以生成专门为 Windows 8 准备的应用程序，并且为了兼容提供了许多特别的功能。然而，如果您希望通过 Windows 应用商店发布这个游戏，您 must 拥有一个 Microsoft Developers Licence (微软开发许可证账号) 可以在 这里 注册这个账号。注册这个账号是为了获取您的有效的银行账户及信用许可以便使得您可以通过您的应用程序赚钱。

更多关于 Windows 8 模块的信息可以在 YoYo Games 服务台 找到。

Mac OS X 模块

使用 Mac OS X 模块，您需要一个拥有 Intel 芯片以及运行 Mavericks (OS X 10.9) 或更高版本的 Mac 电脑。你还需要下载并安装您的操作系统版本的Xcode (至少 Xcode 版本 6)。. 通常建议您使用最新的OS X和新的Xcode (您可以点击顶部左侧的 "苹果" logo 并选择 "关于MAC" 找到您的MAC版本)。

更多关于 Mac OS X 模块的信息可以在 YoYo Games 服务台 找到。

Ubuntu (Linux)

Linux 模块用于创建 Ubuntu 操作系统程序，尽管它也可以运行在其他 Linux 系统中，但是建议运行在最新最流行的Ubuntu Linux 操作系统中。

更多关于 Ubuntu 模块的信息可以在 YoYo Games 服务台 找到。

iOS

GameMaker: Studio iOS 模块需要您有一台 Mac 电脑 (或至少使用一台) 并且符合 MacOSX 的最低标准 (如上)。您必须注册苹果开发者并将所有证书和许可更新。更多关于注册苹果开发者和如何获得所需的证书请访问 苹果开发者网站

几乎所有IOS设备都兼容 GameMaker: Studio，它们包括:
  • iPhone 4、4S、5、5C、5S、6 和 6plus
  • iPad 1 以及更高
  • iPod Touch 3rd Gen 以及更高

对于用 GameMaker: Studio 建立的所有iOS项目，不论是测试 iPhone 还是 iPad 都没有关系，因为它们是标准的、通用的应用。这意味着所有的iPhone和ipad均可运行您的游戏。

更多关于 iOS 模块的信息可以在 YoYo Games 服务台 找到。

Android

开始使用 GameMaker: Studio 制作安卓程序之前，要注意安卓设备差异极大。GameMaker: Studio 制作的游戏仅能运行在 Android 2.3 (Gingerbread) 及其以上版本 (API Level 9 以上的Android SDK)，涵盖了目前大多数设备，但不是所有设备。所以请确认您使用的 Android 版本。并且建议您的设备最好有一个专门的 GPU OpenGL 1.1 或以上版本。 没有 GPU 的设备仍然可以运行您的 GameMaker: Studio 项目，但预计性能会严重下降。.

您还需要下载 GameMaker: Studio 不同版本的 Android SDK 组件和 Platform Tools。您可以在 here 获得SDK。. 如果您使用 YoYo Compiler (YYC) 您将需要 Android NDK ，可以在 here 获取。

更多关于 Android 模块的信息可以在 YoYo Games 服务台 找到。

HTML5

HTML5 扩展模块只需要一个支持 HTML5 的浏览器。注意尽管如今的浏览器均可以运行HTML5游戏，但是并不是所有的浏览器都支持 WebGL 标准。所以我们建议使用 Chrome 测试以便显示模块所支持的所有功能。
建立 HTML5 模块的更多信息可以在 YoYo Games 服务台 找到。

Windows Phone

创建 Windows Phone 模块游戏需要安装Windows Phone SDK 8.0 (点击 这里 下载)。您还需要一个具有以下配置的windows 8 电脑。:
  • 64 位 Windows 8 系统。32 位 系统不支持。
  • 最小 4GB 内存
  • •6.5GB 磁盘空间

如果您没有 Windows Phone 设备，您可以使用 Windows Phone 模拟器，但是注意这需要额外的需求:

  • Windows 8 专业版或企业版
  • 硬件虚拟化
  • 二级地址转换 (SLAT)
  • •DHCP，使得虚拟机可以获得本地IP地址

注意: : Windows Phone SDK 还将安装 Microsoft Visual Basic Express for Windows Phone ，因此无需单独安装它。

更多关于 Windows Phone 模块的信息可以在 YoYo Games 服务台 找到。.

Tizen

Tizen 模块具有一定的先决条件，其中最重要的是必须安装Tizen SDK (点击 这里 获取 SDK)。

。警告!:! 当前的 Tizen SDK 不能在 Windows 8 上工作，所以你需要在你的电脑上安装早期的系统 (你可以安装 Windows XP、Vista 或 win7)。还需注意 64 位 SDK 可能也 无法工作，甚至如果你的系统是 64 位，所以尽可能选择 32 位版本。

此外你还需要:
  • 双核 2 GHz CPU
  • 至少 2G 内存
  • 至少 3G 磁盘空间
  • •安装 Oracle Java JDK (v6 或以上) (not 使用 OpenJDK)

如果你不能访问 Tizen 设备，你也可以使用 Tizen 模拟器，然而你必须满足以下需求以保证正常工作:

  • 支持 Intel VTx 的 CPU
  • 至少 1280 x 1024 的屏幕分辨率
  • 用于 OpenGL ES 加速的最新显卡驱动
更多关于 Tizen 模块的信息可以在 YoYo Games 服务台 找到。

The YoYo Compiler (YYC) 模块

YYC 使得普通的 GameMaker: Studio 输出并编译为目标平台的代码，“剥离” 不需要的函数，进行一系列技术优化，然后创建一个更小的、性能更强的执行程序。这可以增加至少两到三倍的游戏性能，尤其是一些逻辑复杂的游戏，更是能获得一个难以置信的整体提升，非常适合那些较大、十分消耗 CPU 的游戏。

虽然 YYC 被划分为单独的模块，但实际上是由一些分离的模块组成，每一个都将适用于一个可用的目标平台。所以，如果你有 iOS 和 Android 的模块，然后又获得了YYC，你会看到新的模块已经添加到您的目标平台列表 - iOS (YYC) 和 Android (YYC)。以后其他模块更新了，YYC也将更新，并为目标平台显示一个额外的 (YYC) 的模块。
注意: YYC 不适用于 HTML5 目标平台, 以及任何基于 JavaScript (JS) 的平台。

对于 YYC 目标模块的要求，取决于你拥有和希望使用的原始目标模块，因为它们在 YYC 正常工作之前就需要被正确的设置。你可以在 YoYo Games 知识库 里找到更多的信息，同时在本手册的个人目标模块 Global Game Settings 页面会作详细描述。

注意: 目前，YYC 仅适用于 Windows、iOS、Android 和 Windows phone 的目标模块，但未来更新将添加更多的目标模块。

GameMaker: Player 应用

无论你拥有 GameMaker: Studio 的哪个版本，都会发现它捆绑了一个叫 GameMaker: Player 的独立应用。. 这个是发布游戏以及玩其它的人制作的游戏的一个平台。一旦你安装了 Studio 和 Player， 你可以单独运行他们，也可以上传你制作的游戏到 Player 平台给其他人玩。

更多关于 GameMaker: Player 平台的信息可以在 YoYo Games 知识库找到。
