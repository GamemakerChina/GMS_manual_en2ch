发布您的游戏

在这里讨论的是如何发布你的作品。

手册前面的部分讲解了如何创建一个游戏，但是当游戏完成时你肯定希望其他人可以玩它。当然你可以导出 *.gmz 资源文件并且让他们使用 GameMaker: Studio 来玩你的游戏，但是这通常并不是你想要的！首先，你不希望别人能够改变游戏或得到你所有的辛勤工作，其次你也希望即使别人没有 GameMaker: Studio 也可以玩你的游戏。所以你想创建一个独立的应用程序或把你的游戏打包，或者上传到互联网作为 HTML5 的网站给大家玩！

创建你的独立游戏

使用 GameMaker: Studio 创建独立游戏十分简单。首先你要在 "target" 里选择到底需要生成什么类型的文件，每一个类型都在 Global game settings(游戏全局设置) 和 Preferences(配置) 里有单独的设置。

一旦选择完毕，你可以点击 File 菜单并选择 Create Executable 按钮 (或者工具栏上相应的按钮):
生成目标程序
两者都会打开保存窗口，在这里你可以选择文件名。点击确定后会生成目标程序，就可以如你所愿地发布了。

注意: 在生成最终文件之前需要清理资源编辑缓存 (点击使用工具栏里的绿色 "扫帚" 图标)来确保生成最终游戏的正确性。

目标平台格式

每个目标平台都有一些特殊的格式:

  • • 如果在 GameMaker: Studio 窗口 target 设置了 "Windows" ，那么就 只能 生成兼容 Windows 的文件。可以是一个安装包、一个独立的 *.exe 程序或者 *.zip 压缩包。导出文件类型在 "保存" 对话框的下面选择，如下图:

  • •如果你选择生成 HTML5，那么会生成一个含有 index.html (默认名称，可更改) 的包含你的游戏的文件夹，并保存到指定位置。你需要将整个文件夹 both 上传到服务器。index.html 网页还可以自定义背景颜色或位置等...但是 HTML 语言必要的知识请查阅 Global game settings (游戏全局设置) 里的 HTML5 Tab (HTML5 选项卡)

  • 选择了安卓设备，就会生成 *.apk 文件，你可以直接在任何 Android 设备上使用，或者上传到 Google Play 或 Amazon stores (或任意接受 Android 应用程序的网站)。

  • 生成 iOS 或 Mac 应用软件需要 苹果开发者许可和一台苹果 MAC 电脑。GameMaker: Studio 将创建 MAC 电脑上 Application Oven 所需的 *.dish 文件。这将生成你的 iOS 或 MAC 应用软件。

  • •Windows 8 (JS 和 本地) 模块会生成 *.appx 文件，还会附带生成一些 win8 app 必须的文件。这些文件都将被储存在你选择的文件夹里。

  • •Windows Phone 模块会生成 *.xap 文件，可在所有兼容设备上运行或发布到 MS Store。

  • •Tizen 模块会生成 *.wgt 文件，可在所有兼容设备上运行或发布到 Tizen 应用商店。

  • •如果你在 Steam 上购买了 GameMaker: Studio 则可以编译到 Steam 创意工坊。更多信息请查阅 这里。

请注意: GameMaker: Studio 使用 C++ 创建最终游戏。老版本的 GameMaker 使用的是 Delphi，这意味着从老版本导入的游戏 可能不会工作，除非你作出正确的修改。

更多如何生成目标平台最终游戏和上传 app 到应用商店的细节请登陆 YoYo 知识库。

发布

一旦你创建了独立应用程序或打包了你的游戏，你就可以把你的文件提供给别人或者发布到网站上供别人下载。你也可以上传这些文件到不同主机服务器或在线商店 (如 Google Play、iTunes 或 MS Store) 发布和销售。
有关这些格式和文件创建的详情请查阅 YoYo 知识库。

你可以用任何你喜欢的方式发布你用 GameMaker: Studio 创建的游戏，这意味着可以卖给别人！当然，前提是所有的游戏资源都是你自己制作的。有关许可的更多信息请查阅 Licences Tab(许可标签))。
