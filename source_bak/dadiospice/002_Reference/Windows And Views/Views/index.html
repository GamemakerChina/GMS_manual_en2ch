视野

本节论述了定义视野的相关函数。

视野是GameMaker: Studio可以操作的显示属性中最重要的一个。视野，基本上是进入游戏世界的小窗口，它们可能是展示房间的一部分，也可能是与房间大小1:1的缩放，因此在房间比显示器更大时视野是必须的。无需代码在房间编辑器内设置视野的更多信息，请查看章节More About Rooms - Views.

GameMaker: Studio 允许8个独立的视野（编号0-7），所有的视野都可以在任何时候激活，而且都可以展示相同房间的不同地方。这意味着可以使用视野绘制HUD界面（抬头显视系统，如血条、金钱），或者分裂屏幕效果。每个视野都有关联的视口（视见区域）. 如果说视野是玩家看见的房间的一部分，而视口是视野绘制在屏幕上的区域，所以如果视野不同于视口时，你可以对视野进行变形或缩放。

关于视野和视口需要注意的是，游戏第一个房间所有活动视口边框总面积 ，便是定义背景画布大小的东西（或者MacOS、Ubuntu（Linux）和Windows的窗口大小），参考下图的说明： 最后，你应该注意多个视野的情况，因为对于 所有实例的绘制事件，每次视野都会调用一次，如果你一个房间有三个活动视野，绘制事件每步都会运行三次（基本上是三倍的运行量），如果游戏太而复杂，这会导致游戏运行速度减慢。不过， view_current 变量可以用来限制绘制的调用。
