physics_fixture_bind_ext

这个函数将一个支架绑定给一个实例，并且按照给定数值来偏移它的x或y位置。

语法:

physics_fixture_bind_ext(fixture, target, xoffset, yoffset)


参数 描述
fixture 待绑定的支架
target 待接收支架的目标实例（可以是实例id、一个物体索引、关键字other或者all）
xoffset 沿x轴偏移量
yoffset 沿y轴偏移量


返回值:实数


描述

一旦我们定义了我们的支架，它就必须被绑定给一个实例。这也就意味着它的属性就被转移到了被选定的实例，而并非真正的支架本身，以便于使一个支架可以被绑定给多个实例如果全体都有相同的属性。你能够指定物体的索引作为目标并且所有同期在房间内出现的这个物体的实例都将接收到这个支架属性（而后来被创建的物体的新实例没有），或者你可以使用特殊的关键字other和all。你甚至可以指定一个父物体然后带有那个父对的所有子物体都会收到这个支架。一旦支架已经被绑定给所有你需要的实例如果不再必要它就可以被立即销毁了，并且具有那个支架属性的实例将不再受到影响并且能够维持那个属性。

正常地，这个支架会被绑定到这个实例位于实例原点的质心处，然而，它经常不是你所要求的位置并且因此这个函数允许你按照被给定的数值偏移支架被绑定的x和y位置（如果你不要求这一点那就请使用physics_fixture_bind来代替）。注意到这个支架只能支持一个单独的偏移，因为在一个支架上添加多个偏移是不被Box2D支持的。 这个函数还将会为这个被绑定的支架返回一个唯一的"id"值，其能够接下来被用于使用physics_remove_fixture函数从实例移除（“解绑定”）物理属性而不需要销毁重建物体。

注意: 支架应当在不再被需要的时候被删除，因为如果没能做这件事将可能导致内存移出，其可能拖慢你的游戏并且最终使你的游戏崩溃。


举例:

var fix, inst;
fix = physics_fixture_create();
physics_fixture_set_circle_shape(fix, 16);
physics_fixture_set_density(fix, 1.0);
inst = instance_create(x, y, genericBodyObject);
my_fix = physics_fixture_bind_ext(fix, inst, sprite_width / 2, -(sprite_height / 2));
physics_fixture_delete(fix);

上述代码将会创建一个支架并且将其索引分配给变量"fix"。之后他又在以基于精灵宽高的偏移量绑定它到一个实例之前定义了支架的形状和密度。绑定好的支架的实例呗存储在变量"my_fix"中、最终支架自身被销毁来防止内存溢出。因为它不再被使用了。
