show_debug_overlay

显示调试外观。

语法:

show_debug_overlay(enable);


参数 描述
enable 开启 (true) 或关闭 (false) 调试外观。


返回值: 无


描述

这个函数可以被用于在测试你的游戏时开启或者关闭标准调试外观并且默认是关闭的。调试外观在实际游戏窗口中显示了一个图像CPU/GPU占用条还有当前的真实fps值、贴图交换数量和顶点批处理的数量（注意贴图交换数量和顶点批处理数量不会是0并且正常地会显示为值2或3，因为即使一个没有物体的空房间,GameMaker: Studio仍旧需要去绘制并且进行批处理）。



这个条被分为许多小节，每一节都是一秒的1/60。正如你能够在上图中看见的那样，这个条由不同的颜色组成：

  • 绿色 - 输入 / 输出进程（换言之：键盘、鼠标、手柄、网络等……）
  • 红色 - 步事件更新速度。
  • 黄色 - 绘制事件要求的时间。
  • 橙色 - 调试更新事件，其通常只在你使用调试模块的时候可见。
  • 白色 - GPU剩余时间，是被消耗于下一帧能开始之前等待GPU完成帧的渲染的时间。
  • 青色 - 文本渲染时间。
  • 灰色 - 每步绘制清空屏幕所需时间。
  • 暗红 - GPU冲洗时间，是GPU用来从内存清除图像所用时间。

使用这个函数你无论是否在调试模式都能添加这个外观，并且通过这种方式，你能够看到你的游戏运行的效率有多高并且得到一个关于它如何使用可用资源的视觉线索，而顶部无需让调试器在旁运行。


举例:

if global.debug
   {
   show_debug_overlay(true);
   }
else
   {
   show_debug_overlay(false);
   }

上述代码将会依据一个全局变量的值来切换调试开启或关闭。
