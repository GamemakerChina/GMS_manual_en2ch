dot_product

返回向量(x1,y1)与(x2,y2)的点积（数量积）。


语法:

dot_product(x1, y1, x2, y2)


参数 描述
x1 第一个向量的x坐标。
y1 第一个向量的y坐标。
x2 第二个向量的x坐标。
y2 第二个向量的y坐标。


返回值: 实数


描述

点积是一个表达两个向量间角度关系并用两个向量相乘①然后相加的结果。“点积”这个名字源自于经常被用于指明这种运算的中心圆点符号“·”（又称为“标量积”（数量积），强调了其结果本质是一个标量而不是向量）。

实质上的数学公式可以被写成像这样：

①：此处的“相乘”表示两个向量的分量各自相乘，即x1y1和x2y2


所以在2维中向量a(x1,y1)和b(x2,y2)的点积是x1y1+x2y2，意味着在GameMaker: Studio中点积被计算为：

a · b = (x1*x2)+(y1*y2);

关于点积最令人好奇的事情就是它与由输入的向量形成的角度的关系，其被表达为：

a · b = |a| × |b| × cos<a,b>
//<a,b>表示向量a与向量b的夹角

这就是说，两个向量的点积将等于那些向量的夹角的余弦，乘以它们各自的模。这里有一张插图：

有一些事情我们能够从任意两个向量有关的点积中阐明：

  • 如果 a 和 b 是相互垂直的（对彼此成90°角），这个点积的结果将是0，因为cos<a,b>会是0.
  • 如果 a 和 b 的夹角小于90°，这个点积就会是正的（大于0），因为cos<a,b>将是正数。
  • 如果 a 和 b 的夹角大于90°，这个点积就会是负的（小于0），因为cos<a,b>将是负数。

那么这对于我们这些做游戏的人意味着什么呢？好吧，这个数学关系可以被用于相当多的场合中，但最好观察它的方式就是去建立一个真正的情形并由你自己来观察发生了什么。做到这一点的最简单方式之一就是去生成一个单独的用于检测敌人是否在内的“视野线”，比方说，一个这样的平台游戏，敌人将会“观测”玩家是否在它们的法向量两边90°范围内。

根本上说，我们需要获取到敌人的法向量（换言之：包含了方向和视野距离）并且接下来我们需要去获得玩家到敌人的向量我们然后将得到这些向量的点积，并且如果结果为正则玩家已经被发现，而如果是负的则没有。来达成这种运作的实际代码被提供于下方的范例中。


举例:

var x1, y1, x2, y2;
x1 = lengthdir_x(1, image_angle);
y1 = lengthdir_y(1, image_angle);
x2 = o_Player.x - x;
y2 = o_Player.y - y;
if dot_product(x1, y1, x2, y2) > 0 seen=true else seen=false;

上述代码使用实例的image_angle创建了一个单位向量，并接下来获取从玩家物体"o_Player"到它自身的向量。最终它计算了这两个向量的点积并且如果大于0就将变量"seen"设定为true，且如果小于等于0则设定为false。
