dot_product_3d

返回三维向量(x1,y1,z1)和(x2,y2,z2)的点积。


语法：

dot_product_3d(x1, y1, z1, x2, y2, z2)


参数 描述
x1 第一个向量的x坐标。
y1 第一个向量的y坐标。
z1 第一个向量的z坐标。
x2 第二个向量的x坐标。
y2 第二个向量的y坐标。
z2 第二个向量的z坐标。


返回值: 实数


描述

点积是一个被表达为两个向量间夹角关系的值并且是通过取得两个向量，相乘在一起然后将结果相加得到的。“点积”这个名字源自于经常被用于指明这种运算的中心圆点符号“·”（又称为“标量积”（数量积），强调了其结果本质是一个标量而不是向量）。

实质上的数学公式可以被写成像这样：


在2D中，向量a(x1,y1)和b(x2,y2)的点积是x1x2+y1y2，3D中也是一样，向量a(x1,y1,z1)和b(x2,y2,z2)的点积是x1x2+y1y2+z1z2。所以dot_product_3d在GameMaker: Studio中被计算为：

a · b = x1x2+y1y2+z1z2;

关于点积最令人好奇的事情就是它与由输入的向量形成的角度的关系，其被表达为：

a · b = |a| × |b| × cos<a,b>
//<a,b>表示向量a与向量b的夹角

这就是说，两个向量的点积将等于那些向量的夹角的余弦，乘以它们各自的模。这里有一张插图：

有一些事情我们现在能够从任意两个向量有关的点积中阐明：

  • 如果 a 和 b 是相互垂直的（对彼此成90°角），这个点积的结果将是0，因为cos<a,b>会是0.
  • 如果 a 和 b 的夹角小于90°，这个点积就会是正的（大于0），因为cos<a,b>将是正数。
  • 如果 a 和 b 的夹角大于90°，这个点积就会是负的（小于0），因为cos<a,b>将是负数。

那么这对于我们这些做游戏的人意味着什么呢？好吧，这个数学关系可以被用于相当多的场合中，但最好观察它的方式就是去建立一个真正的情形并由你自己来观察发生了什么。做到这一点的最简单方式之一就是去生成一个单独的用于检测敌人是否在内的“等高线”，比方说，一个这样的平台游戏，敌人将会“观测”玩家是否在由敌人的法向量和3D地面形成的平面的上方。

根本上说，我们需要获取到敌人的基向量①（换言之：包含了方向和视野距离）并且接下来我们需要去获得玩家到敌人的向量我们然后将得到这些向量的点积，并且如果结果为正则玩家已经被发现，而如果是负的则没有。来达成这种运作的实际代码被提供于下方的范例中。

①：此处基向量即单位向量，而非基底向量。


举例:

var x1, y1, x2, y2;
x1 = 0;
y1 = 1;
z1 = 0;
x2 = o_Player.x - x;
y2 = o_Player.y - y;
z2 = o_Player.z - z; if dot_product_3d(x1, y1, z1, x2, y2, z2) > 0 above=true else above=false;

上述代码沿着实例的y（向上）轴创建一个基向量，并且接下来获取玩家物体"o_Player"到它自身的向量。最终它计算了这两个向量的点积并且如果大于0就将变量"seen"设定为true，且如果小于等于0则设定为false。
