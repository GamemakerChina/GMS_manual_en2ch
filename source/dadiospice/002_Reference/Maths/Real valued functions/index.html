Real Number Functions 实数函数

The following functions exist that deal with real numbers.
下面存在一些函数来处理实数。

Real numbers in GameMaker: Studio are considered floating point numbers, that is to say that they have a decimal point in them with no fixed number of digits either before or after the point. 2, for example, is a binary integer but 2.0 is a floating point real. This distinction between integers and floats is very important when dealing with cross platform development as the precision of calculations made on a Windows pc is not the same as the precision of those same calculations when made on a mobile device. This means that you should pay particular attention when making comparisons, for example:
在GameMaker: Studio中，实数被认为是浮点数， 也就是说它们中有小数点带有不固定的数字要么在小数点前要么在小数点后。 比如， 2， 是一个二进制整数， 但在浮点数中实际是2.0。 整数与浮点数这个差别在处理跨平台开发的时候是非常重要的, 因为在Windows PC上计算的精度与在移动设备上同种计算的精度是不同的。 这就意味着你在比较的时候应该特别注意， 比如说：

if image_index == 1 {do something...}


Now that may appear to work, but due to the way that floating point maths works, it may mean that image_index actually equals 1.00000000001, in which case the comparison will never result in a true result and the code running. These types of errors can be quite hard to debug and so it is always good to be aware of them and to plan ahead and use other means of checking values or to use the appropriate flooring or rounding functions (listed below) to convert the number to check into an integer. 现在这可能会生效， 但是因为浮点数学的工作原理， 这可能意味着image_index实际上等于1.00000000001， 在这种情况下比较将永远不会导致一个true结果以及代码的运行。 这种错误是相当难调试的并且因此经常小心它们是非常好的并且要预先计划以及使用其他检测值的方法或者使用相应的向下取整或者四舍五入函数（在下方已列出）来把实数转化成整数。 比如说上述代码就可以被写成：

if floor(image_index) == 1 {do something...}

我们能够在GameMaker: Studio中使用一个特殊的可用的函数来为浮点数学设定ε值（精度无穷小值）。 当一个实数被取舍到了最近的浮点数， 这个ε（也叫做“机器Epsilon”）形成一个相对误差的上界， 并且你能使用这个特殊的函数来设置ε值。

  1. math_set_epsilon

Random Functions 随即函数

These functions all deal with using random numbers and values:
这些函数全部都是处理随机数和值的：

  1. choose
  2. random
  3. random_range
  4. irandom
  5. irandom_range
  6. random_set_seed
  7. random_get_seed
  8. randomize

NOTE 注意: When using the random functions, GameMaker: Studio maintains the same random seed every time you start the game. This makes debugging much easier as you are guaranteed that the random functions will always return the same value, however should you not wish this to happen, you must first set a new random seed at the very start of the game, either using randomize or random_set_seed.
当使用这些随机函数时， GameMaker: Studio每次你开始游戏都保持同一随机数种子。 这使得调试更加简单因为你保证了随机函数总是将返回相同的一套数值， 然而你也不愿意这件事情发生， 你必须在每次游戏开始的时候首先使用一个新的随机数种子， 要么用randomize函数要么用random_set_seed函数
