使用 3D

本章节详细介绍用于创建3D游戏的函数.

GameMaker: Studio 主要用于 2D 游戏的开发,同时,你也能够使用这些函数用于 3D 游戏的创作. 在开始之前,你需要了解 GMS 支持 3D 的情况:

  • 提供的 3D 函数局限于绘制方面.而其他方面,需要你自己去实现或者使用别人写好的的扩展,比如碰撞检测,深度排序.只要你有足够的 3D 编程知识以及 GMS 使用经验,是能够创造出任何的3D功能和效果的。

  • 在3D模式下，GMS提供的一些功能可能不再适用。
    1. 房间提供的背景以及前景属性。两者用于平铺图像，在3D透视投影下，将无法取得预想的效果。
    2. 获取正常的鼠标坐标。你依然能够获得鼠标坐标，但坐标被映射为当前屏幕或者视野的鼠标坐标。如果需要，你必须进行坐标转换。
    3. 图块以及碰撞检测。图块可能无法取得预想的效果。而碰撞检测是基于房间内的二维坐标系，在某些情景下你仍然可以使用碰撞检测，比如俯视角的赛车游戏，但无法用于检测三维坐标系中的碰撞。
  • 所有的3D函数，没有提供按钮，需要你自己去编写代码，熟悉 GML 是下一步的前提。

  • 你必须具备3D图形的基础知识,本教程不会详细描述相关名称所代表的含义，比如透视投影，表面剔除，灯光以及雾。 （你可以在DX或者OpenGl相关的书籍中学到3D图形的基础知识）
