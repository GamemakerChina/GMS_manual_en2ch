d3d_start

设置 GameMaker: Studio 之后的绘制都采用 3D 模式。

语法：

d3d_start()


返回：: 实数


说明

这个函数可以在任何事件中调用，并影响随后的绘制效果。如果你的游戏不是完全由 3D 绘制组成，你可以随时调用d3d_end，执行剩余的 2D 绘制。

开启3D模式后，以下部分会发生变化：

  • 第一点，深度测试（也叫做 z-buffer）被开启，而GMS支持 24-bit 的 z-buffer。 这意味着，屏幕上的每一个坐标，如果存在多个像素点，只绘制深度值（z-value）最小的那个像素点。 如果这几个像素点的深度值相同，那么这几个像素点都会被绘制到屏幕上，导致丑陋的画面效果。 人们把这个问题称作深度冲突（Z-fighting），要解决这个问题，尝试确保实例间的深度不要重叠。

  • 第二点，默认的正交投影被透视投影取代。这意味着，呈现在屏幕上的物体大小随深度发生变化，深度越大， 大小越小。物体的深度为0，那么呈现出大小为原始大小（除非你做了修改）。 摄像机的位置在房间上方（距离等于房间的宽度）,只有处于摄像机前方的物体（深度大于0）才会被绘制。因此不要绘制深度小于0，或者说，不要小于-w（w 表示房间或者视野的宽度）。

  • 第三点，3D 模式下的坐标系，Y轴正方向相对屏幕向上（2D 模式下，Y轴正方向相对屏幕向下）。同时，相对屏幕的坐标（0，0），位于屏幕的左下角


示例：

d3d_start();

这段代码放在 create 事件，设置 GameMaker: Studio 之后的绘制都采用 3D 模式。
