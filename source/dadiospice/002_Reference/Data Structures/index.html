Data structures数据结构

This section deals with the different, specialist functions that you can use to store different types of data.
这一节处理不同的、特殊的你能够用于存储不同类型数据的函数

In games you often need to store information in a precise and ordered way. For example, you may need to store lists of items that a person carries or you may want to store a grid of places that still need to be visited. Now, you can use the arrays for this, but if you want to do more complicated operations, like sorting the data or searching for a particular item, you need to write large pieces of GML code which can be slow to execute, difficult to debug and a real pain to write!
在游戏中你通常需要以一个精确的、有序的方式存储信息。比如说，你可能需 要存储一个人物携带物品的列表或者你可能想要存储一个仍然需要被访问的地 点的栅格。现在你能够使用使用数组来做这件事情，但是如果你想要执 行更复杂的操作，比如分类这些数据或者搜索一个特定的项目你需要写一大篇 会拖慢执行的、难以调试的并且写起来十分痛苦的GML代码！

To remedy this, GameMaker: Studio has a number of built-in data structures that can be accessed through specialist functions. There are six different types of data structure available, each one having its own benefits depending on the type of information that you are looking to store and how you wish to manipulate it later: stacks, queues, lists, maps, priority queues, and grids.
为了改进这个，GameMaker: Studio有许多内置的数据结构能 够通过特殊的函数被访问。有六种不同类型的数据结构是可用的，每一个都有 取决于自己的优势，这取决于你打算存储信息的类型以及你接下来如何分配： 堆栈、队列、列表、比对、优先队列、栅格。

Essentially, all data structures work in the same way - You create a data structure and store its index value in a variable. You then use this index to reference the data structure in all further function calls that can perform operations on it. Finally, once you are done you destroy the data structure again to remove it from memory. You can use as many of the structures at the same time as you need, and all structures can store both strings and real values.
必要地，所有的数据结构工作方式都是同理的——不成功押宝规划部宇哥数据结 构并且把他的索引值存入一个变量。之后，你就可以用这个索引表示 该数据结构，以便对其进行操作。最终，一旦你完成了，你就又销毁这个这个 数据结构并将其移出内存。你可以同时使用你所想要的尽可能多的结构并且所 有结构都既能存储字符串又能存储实数

NOTE注意: As with all dynamic resources, data structures take up memory and so should always be destroyed when no longer needed to prevent memory leaks which will slow down and eventually crash your game.
由于伴随所有的动态资源，数据结构将会占用内存并因此应 该在不再被使用的时候被销毁以防止内存泄漏将拖慢你的游戏并且最终使你的游戏崩溃
